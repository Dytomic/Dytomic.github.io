Role & Scope

CTO and engineering direction, end to end

Owned technical direction for an independent game studio building a PC MMORPG from first principles on top of a legacy engine.

Dytomic was founded to ship Pirate Raids Online — a PC MMORPG drawing from the classic Tales of Pirates / Pirate King Online (PKO) engine, reimagined with modern design. As CTO I led every technical decision from engine choice through to the public web presence.

The remit covered: hiring and running the engineering team, reverse-engineering the TOP/PKO engine to understand what we were building on, selecting and prototyping the server stack (we explored Rust and Go rewrites of the GateServer), running the full web marketing stack (Next.js), and running the live technical interview process for game-developer candidates.

Stack decisions

What was chosen, and why

Every layer of the stack was picked deliberately, with trade-offs documented in the repos themselves.

Rust for server rewrites

The legacy GateServer is a single-threaded Windows-only binary. Rewriting in Rust gave us Linux deploys, multi-region redundancy, and safe concurrency without rewriting the game logic.

Go for comms plane

A Go interface layer for direct communication between GateServer, GameServer, and GroupServer — picked for fast iteration and solid networking primitives.

Next.js 12 + React 18

Marketing site on Next.js 12 with React Bootstrap and CSS Modules. SSR for SEO, static export-friendly, fast enough to keep page weight low on a content-heavy lore page.

C++ for client mods

The engine is a Win32 C++ codebase. Client modifications use the PKOdev mod loader — a shared-library injection system — compiled against the same MSVC toolchain as the original.

Laravel for player CMS

Private-server CMSes were pragmatic choices: Laravel for the full player portal with auth / rankings / purchases, and a separate JS CMS for operational tooling.

Python for ops bots

Ariel, the Facebook Messenger intake bot, was a deliberate single-file Python / Flask app. The README’s own words: a reminder not to over-engineer relative to the value delivered.

Deep technical work

Tales of Pirates & PKO engine research

Pirate Raids Online sat on top of a legacy MMORPG engine. Shipping a modern game meant understanding that engine deeply — these two monorepos are the preserved research archive.

Both repos are git-subtree mirrors of the full public bodies of work by community engine developers V3ct0r1024 and Perseus. They’re maintained here as syncable upstream references — each subtree can be pulled independently as the original authors publish changes.

pkodev-V3ct0r1024 →

42 upstream repos · Client + Server
C++Win32Shared library injectionLua

42 open-source Tales of Pirates mods and tools by V3ct0r1024, mirrored as git subtrees. Contents: the pkodev.mod.loader shared-library mod loader, client-side mods (60fps uncapping, anti-bot, full-map reveal, item-level icons, classless stat systems), server plugins (daily reward chains, player rating, GM command tools, mob respawn fixes), and the offline stall server (C++/Winsock).

pkodev-Perseus →

6 upstream repos · Infrastructure
RustGoPHP 8LaravelJavaScript

Perseus’s server infrastructure work, mirrored as git subtrees. Contents: top-server-rewrite (full Rust rewrite of the GateServer), top-proxy (Rust middleman proxy providing DDoS filtering and Linux deployability without touching the GateServer), top-comm (Go comms interface), pko-tools (Rust asset converter to/from glTF for Blender workflows), and two private-server CMS projects on Laravel and JavaScript.

Context

What the game was

Pirate Raids Online (PRO) was the product around which all of this engineering happened. Brief context for readers coming to this page from a technical angle.

Pirate Raids Online — an old-school inspired PC MMORPG set in the world of Ascaron. Players took the role of a Demigod leading The Pirate Coalition in a rebellion against The Navy. Four playable Demigod tribes (Dwarves, Triton, Runeborne, Amazonians), classless combat with up to 10 weapon builds, open world PvP, naval combat with player-built ships, and old-school systems (guilds, life skills, player-driven economy).

The full marketing-side vision lives in the pro-web repo. The studio shut down in early 2024; everything here is archived as a public technical record of the work.
